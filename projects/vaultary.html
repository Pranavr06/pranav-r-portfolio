Vaultary

Identity & Security Python • Flask • Supabase • OAuth By Pranav R

1. Executive Summary

Flask (Backend) Supabase (PostgreSQL) OAuth 2.0 AES-256

Vaultary is a production-ready Identity Management System built to solve the issue of credential theft. Unlike standard password managers, Vaultary employs a Zero-Knowledge Architecture, ensuring that the server never sees the user's raw master password. The system integrates seamless login via Google and GitHub and enforces security through Time-based One-Time Passwords (TOTP).

2. Security Architecture

The core philosophy of Vaultary is "Trust No One" (Zero Trust). The security model is built on three pillars:

2.1 Zero-Knowledge Encryption

User data is encrypted using AES-256 (Advanced Encryption Standard). The encryption key is derived from the user's Master Password, which is never stored in the database. Even if the Supabase database is compromised, the attacker only sees encrypted gibberish.

2.2 Authentication Flow

We moved away from storing local passwords to using OpenID Connect (OIDC). This delegates trust to major providers, reducing the attack surface.

  • Primary Auth: OAuth 2.0 via Google and GitHub.
  • Secondary Auth: TOTP (Time-based One-Time Password) using pyotp, compatible with Google Authenticator.

3. Technical Challenges & Report

During the deployment phase to Vercel (Production), several critical engineering challenges were encountered. Below is the technical report on how they were resolved.

ISSUE #1: OAuth Protocol Mismatch (Vercel Proxy)

Status: ✅ Resolved
Description: Login worked on `localhost` but failed on Vercel with a `500 Internal Server Error` / `MismatchingStateError`.

Root Cause: Vercel's serverless functions sit behind a reverse proxy. The internal request logic in Flask detected `HTTP` traffic, generating a callback URL like `http://vaultary...`. However, OAuth providers (GitHub/Google) strictly require `HTTPS`.

The Fix: We had to explicitly force the URL scheme generator to ignore the internal protocol and enforce HTTPS.

# app.py - Fix Applied url_for('github_callback', _external=True, _scheme='https')

ISSUE #2: LinkedIn OAuth API Failure

Status: ⚠️ Deprecated / Fixed via Removal
Description: LinkedIn authentication resulted in persistent `client_secret` missing errors despite correct environment variables.

Technical Decision: To prioritize system stability for the final release, the LinkedIn module was deprecated. The system now relies exclusively on the more stable Google and GitHub OIDC implementations. This decision improved overall system uptime and reduced dependency overhead.

ISSUE #3: Database Connection Handling

Status: ✅ Optimized
Description: Serverless functions spin down rapidly, leading to potential database connection drops.

Solution: Implemented a connection pooling strategy with Supabase (PostgreSQL) and ensured that `psycopg2` handles cursor closures gracefully within the Flask application context.

4. Features & Analysis Tools

Beyond storage, Vaultary actively analyzes security hygiene.

Entropy Analysis

We utilize the zxcvbn algorithm to calculate password entropy (bits of randomness). This detects patterns humans often think are clever but computers guess easily (e.g., "P@ssword123").
